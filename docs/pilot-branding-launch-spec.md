Commonshub — Pilot Branding & Launch Spec (v2)

Purpose

This document defines the minimum assets, identity, and content required to launch a production-quality pilot community.

It is designed for:
	•	fast white-glove onboarding
	•	consistent visual quality
	•	repeatable setup for future guided flows

This is not a full customization system.
This is the 90-minute launch kit.

⸻

1. Creator Asset Request (External)

Send this to the creator.

What we need from you
	1.	Community / Show Name
	2.	Desired subdomain
Example: community.indicationsandwarnings.com
	3.	Square logo
Preferred: 400 × 400 px or larger
	4.	Header / banner image (optional)
If not provided, we will generate one.
	5.	Short description (1–2 sentences)
What this community is for.

That’s it.

We handle everything else.

⸻

2. Visual Asset Specifications (Internal)

2.1 Server Header

Primary visual identity.
	•	Size: 1500 × 500 px
	•	Safe area: 1200 × 320 px centered
	•	Format: PNG or JPG

⸻

2.2 Server Thumbnail / OpenGraph Image

Used for:
	•	link previews
	•	server directories
	•	invite links
	•	social sharing
	•	Size: 1200 × 630 px
	•	Safe area: 960 × 500 centered
	•	Format: PNG or JPG

Required for every pilot.

⸻

2.3 Instance / System Avatar

Used for:
	•	admin account
	•	system voice
	•	Size: 400 × 400 px

⸻

2.4 Creator Avatars

For hosts.
	•	Size: 400 × 400 px

⸻

2.5 Favicon / App Icon
	•	Size: 512 × 512 px PNG

⸻

3. Identity Settings

3.1 Server Display Name

Format:

{Show Name} Community

⸻

3.2 Short Description

Target: 120–160 characters

Appears on:
	•	homepage
	•	previews
	•	directories

⸻

3.3 Extended Description (About Page)


Purpose: Immediately communicate what this space is and how to participate.

Target: 500–900 characters
Hard max: ~1,200 characters

Canonical structure
	1.	What this community is
	2.	Who it is for
	3.	Participation tone
	4.	Link to primary platform

Example (~720 chars)
This is the companion discussion community for Indications & Warnings — a chronological, signal-first space for serious geopolitical analysis.

It’s for viewers who want depth, context, and direct engagement, not algorithm-driven noise.

Expect a professional tone, evidence-based discussion, and respectful disagreement.
This is an open, public environment. Read first, then join the conversation.

Main show: youtube.com/…

⸻

3.4 Primary Hashtag

Used for:
	•	discovery
	•	ongoing conversation rhythm
	•	creator posts

Example:

#IndicationsAndWarnings

⸻

4. Default Content to Seed

This is not episode threads.

This is the beginning of a single, ongoing conversation.

⸻

4.1 Welcome Post (Pinned)

Must:
	•	set the tone
	•	explain what this space is
	•	tell people how to participate

⸻

4.2 First Creator Post

A simple post that:
	•	welcomes early members
	•	invites introductions
	•	models the desired style of conversation

⸻

4.3 Seed Community Activity

Minimum:
	•	3–5 thoughtful posts
	•	2–3 replies between test accounts
	•	visible interaction in the local feed

Goal:

Make the space feel alive and safe to enter.

⸻

4.4 Follow Recommendations

Minimum:
	•	hosts
	•	show account
	•	5–10 aligned accounts

⸻

5. Launch Readiness Checklist

A pilot is presentable when:
	•	branding applied
	•	About page complete
	•	rules in place
	•	welcome post pinned
	•	ongoing conversation visibly started
	•	test accounts visible in local feed

No empty timelines.
No visible “default Mastodon”.

⸻

10. Definition of Success

The creator sees the demo and says:

“This already feels like our space.”

	•	recognizable identity
	•	visible conversation
	•	clear tone
	•	no rough edges
