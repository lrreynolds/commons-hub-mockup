# Commonshub Economic & Funding Model Specification
Version 1.0  
Status: Authoritative  
Depends on: commonshub-constitution-v1.0.md  

---

# 1. Purpose

This document defines how funding, membership, and economic alignment operate within Commonshub.

It translates the Constitution’s principles into operational rules.

---

# 2. Core Economic Philosophy

## 2.1 Sustainability Without Extraction

Commonshub exists to enable financially sustainable communities without:

- Advertising
- Surveillance monetization
- Engagement amplification incentives

Revenue must align with:

- Creator sustainability
- Community health
- Infrastructure stability

---

# 3. Default Model (MVP)

## 3.1 Open Community + Optional Funding

By default:

- Participation is always free.
- Community funding is optional.
- Funding does not gate access.
- Funding supports hosting, moderation, and long-term sustainability.

Required language on funding surfaces:

- "Participation is always free."
- "Community funding is optional."

---

# 4. Creator-Defined Access Models (Future Capability)

Commonshub supports creator sovereignty.

Creators may choose:

- Open community (default)
- Hybrid model (free + paid tiers)
- Gated community (membership required)
- Paid access to specific features (livestreams, video archives, etc.)

Commonshub does not mandate one model.

---

# 5. Gated Mode Requirements

If a creator chooses gated access:

Language must clearly state:

- Access requires active membership.
- Membership billing terms.
- Billing is handled by Stripe (or payment processor).

No:

- Hidden walls
- Trial bait-and-switch
- Implicit obligation framing

Clarity is mandatory.

---

# 6. Membership Lifecycle (High-Level)

## 6.1 States

- Not a member
- Active member
- Canceled (grace period)
- Expired

## 6.2 Cancellation Rules

- Clear cancellation path
- No dark patterns
- No forced retention loops

---

# 7. Platform Revenue Model

Commonshub revenue may come from:

- Hosting fees
- Platform subscription tiers
- Transaction fees (transparent)

Commonshub must not monetize:

- User data
- Ads
- Engagement ranking

---

# 8. Announcement & Promotion Patterns (MVP)

## 8.1 Standard Funding Announcement Format

Support this community if you’d like.
Participation is always free. No ads. No algorithms.
Visit → example.community/support

Rules:

- Maximum 3 lines preferred
- No hype tone
- No urgency pressure
- No emoji
- No marketing framing

If gated:

Language must clearly state membership requirement.

---

# 9. Funding Flow Surfaces

Funding must integrate into:

- Community funding page
- Dashboard configuration
- Admin announcement creation
- Pinned post template generation

If funding options change:

- Announcement template should be regenerated
- Pinned post template should be regenerated

---

# 10. Stripe Platform Architecture (MVP)

- Stripe Connect platform account
- Creator-level Stripe accounts
- Platform does not store card data
- Stripe handles billing compliance

---

# 11. Roadmap: Advanced Economic Features

Future possibilities:

- Tiered access control
- Paid livestream access
- Paid video libraries
- Community-level access gating
- Bundle models

These must comply with Constitutional principles.

---

# 12. Non-Negotiables

Commonshub will never:

- Force creators into one economic model
- Require gating as a policy
- Default communities into paywalls
- Use ads to subsidize hosting

Economic freedom is core to creator sovereignty.
